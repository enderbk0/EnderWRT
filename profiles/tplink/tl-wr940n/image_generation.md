# TP-Link TL-WR940N Firmware Image Generation

OpenWrt's buildroot creates multiple firmware image variants to accommodate different installation methods. This document explains the internal partition layout and how the tools pack these binaries.

---

## 💾 Hardware Flash Layout (4 MB total)

The TL-WR940N flash partitioning is defined in the DTS and driver configs. The total 4 MB space is divided as follows:

```
  0x000000 ┌────────────────────────────────────────┐
           │          U-Boot Bootloader             │ 128 KB
  0x020000 ├────────────────────────────────────────┤
           │         Product Info (Model/MAC)       │ 64 KB
  0x030000 ├────────────────────────────────────────┤
           │                                        │
           │         Kernel Image (uImage)          │ ~1.2 MB
           │                                        │
  0x140000 ├────────────────────────────────────────┤
           │                                        │
           │         Root Filesystem (SquashFS)      │ ~2.5 MB
           │                                        │
  0x3F0000 ├────────────────────────────────────────┤
           │         ART (Wireless Calibration)     │ 64 KB
  0x400000 └────────────────────────────────────────┘
```

## ⚙️ Image Types Generated

The compilation output generates the following specific files in the `bin/targets/ath79/tiny/` directory:

### 1. Sysupgrade Firmware (`*-sysupgrade.bin`)
*   **Purpose**: Upgrading an existing OpenWrt/EnderWRT router to a newer version.
*   **Contents**: Contains the Linux kernel (lzma-compressed) and the SquashFS root filesystem concatenated together. It does not contain any TP-Link headers or bootloaders.
*   **Flashing Method**: Upload via the LuCI sysupgrade page, or run `sysupgrade -v /tmp/firmware.bin` via SSH.

### 2. Factory Firmware (`*-factory.bin`)
*   **Purpose**: Flashing a router running the original stock TP-Link firmware.
*   **Contents**: Wraps the sysupgrade binary with a special proprietary header. The header contains:
    *   TP-Link hardware ID (`0x94000006` for v6).
    *   Hardware revision number.
    *   Vendor signature checks.
    *   Padding to align boundaries.
*   **Flashing Method**: Upload directly through the stock TP-Link Web Management Page under "System Tools" -> "Firmware Upgrade".

### 3. TFTP Recovery Firmware (`wr940nv6_tp_recovery.bin`)
*   **Purpose**: Recovering a bricked device or flashing via bootloader.
*   **Contents**: This image is binary-identical to the **Factory** firmware, as U-Boot requires the TP-Link headers to perform basic verification before flashing.
*   **Naming Rule**: Generated by copying the compiled factory image to `wr940nv6_tp_recovery.bin`.

## 🛠️ Automated Packaging Commands

In the OpenWrt Buildroot, image generation is handled automatically using the `mktplinkfw` tool:

```bash
# How Buildroot generates the Factory header
mktplinkfw \
  -H 0x94000006 \
  -V 1 \
  -F 4M \
  -k openwrt-ath79-tiny-kernel.bin \
  -r openwrt-ath79-tiny-rootfs.bin \
  -o openwrt-ath79-tiny-tplink_tl-wr940n-v6-squashfs-factory.bin
```
The script `scripts/build.sh` packages these files, makes copy-renames for TFTP recovery, and places them in `publish_artifacts/`.
